# Google AdSense 승인 도우미 - 작업일지

## 프로젝트 개요
- **프로젝트명**: Google AdSense 승인 도우미
- **목적**: 사용자가 URL을 입력하면 사이트를 분석하여 AdSense 승인 가능 여부를 판단하고, 단계별 개선 솔루션을 제공하는 웹앱
- **기술 스택**: Next.js 16 + TypeScript + Tailwind CSS + Cheerio
- **GitHub**: https://github.com/myjun090-spec/googleadsense
- **배포**: Vercel

---

## 작업 이력

### 1차: 초기 구현 (커밋: fa962b9)
- Next.js 프로젝트 생성 (TypeScript + Tailwind CSS)
- 분석 API (`/api/analyze`) 구현 — 20개 항목 체크
  - HTTPS, 타이틀, 메타 디스크립션, 콘텐츠 양, 네비게이션
  - 개인정보 처리방침, 이용약관, 소개, 연락처 페이지
  - 이미지 ALT, 모바일 viewport, H1 태그
  - robots.txt, sitemap.xml, 광고 과다, 페이지 크기
  - 저작권 표시, 커스텀 도메인, Open Graph, Canonical
- 프론트엔드 3개 탭 구성
  - 분석 결과 (점수 게이지 + 미통과 항목 + 해결 방법)
  - 승인 가이드 (8가지 핵심 조건)
  - 거절 사례 & 해결법 (6가지 거절 사유 + 체크리스트)
- GitHub 레포 생성 및 푸시

### 2차: 분석 정확도 개선 (커밋: d19ee70)
- **한국어 콘텐츠 측정 개선**: 단어 수 → 글자 수(공백 제외) 기준으로 변경
- **필수 페이지 URL 직접 확인**: `/privacy`, `/about`, `/contact`, `/terms` 등 일반적인 경로를 HEAD 요청으로 직접 확인
- **키워드 확장**: "정보처리", "정보보호", "프로필", "운영자", "고객센터", "1:1", "면책" 등 한국어 표현 추가
- **무료 도메인 목록 확장**: 5개 → 19개 (`vercel.app`, `netlify.app`, `notion.site`, `pages.dev` 등)
- **네비게이션 셀렉터 확장**: `.gnb`, `.lnb`, `[role='navigation']`, `#header` 등 추가
- **분석 한계 안내**: 결과 하단에 한계점 명시 (메인 페이지만 분석, SPA 감지 불가 등)

### 3차: 심층 분석 기능 추가 (커밋: 4a55a82)
- **API 전면 재작성** — 20개 → 27개 항목으로 확대
- **사이트맵/RSS 크롤링**: sitemap.xml, sitemap index, RSS 피드를 파싱하여 총 글 수 파악
- **다중 페이지 콘텐츠 분석**: 사이트맵에서 최대 10개 글을 샘플링하여 개별 글자 수, 소제목 수, 이미지 수 분석
- **Google PageSpeed Insights API 연동**: 모바일 성능/SEO/접근성 점수, FCP/LCP/CLS 메트릭 실측
- **Wayback Machine API 연동**: 도메인 최초 아카이브 날짜 → 운영 기간 추정
- **정책 위반 키워드 스캔**: 성인/도박/약물/무기/해킹 관련 키워드를 메인 + 하위 페이지에서 스캔
- **내부 링크 구조 분석**: 메인 페이지의 내부 링크 수/고유 링크 수 측정
- **가중 점수 계산**: 기존 단순 비율 → 필수=5점, 중요=3점, 권장=1점 가중치 적용
- **프론트엔드 심화 분석 UI 추가**:
  - 4개 핵심 지표 카드 (발견된 페이지 수, 도메인 나이, PageSpeed, 정책 위반)
  - 개별 글 분석 테이블 (제목, 글자수, 소제목, 이미지, 판정)
  - PageSpeed 상세 정보
  - 정책 위반 경고

### 4차: 버그 수정 + 예외 처리 (커밋: 3ac8ecd)
- **AdSense 연동 불가 플랫폼 감지**:
  - 네이버 블로그/카페, velog, Medium, 브런치, Notion → 즉시 "연동 불가" 안내
  - 티스토리는 AdSense 연동 가능하므로 정상 분석 유지
- **사이트맵 없을 때 자동 PASS 버그 수정**: 콘텐츠 품질/구조 항목이 사이트맵 없으면 FAIL 처리로 변경
- **도메인 나이 정확도 개선**: Wayback Machine CDX API로 최초 아카이브 정확 조회 + fallback
- **PageSpeed 조건부 체크**: API 연결 불가 시 체크 항목 자체를 생략 (불필요한 PASS 방지)

## 알려진 한계

1. **메인 페이지 HTML만 분석** — 하위 페이지는 사이트맵 기반 샘플링
2. **SPA(React/Vue 등) 감지 한계** — JavaScript 렌더링 콘텐츠는 분석 불가
3. **콘텐츠 독창성 판단 불가** — 복사/AI 생성 여부는 외부에서 측정 불가
4. **실제 트래픽 측정 불가** — 방문자 수는 사이트 소유자만 확인 가능
5. **PageSpeed API 속도 제한** — 무료 API이므로 요청이 많으면 실패 가능
